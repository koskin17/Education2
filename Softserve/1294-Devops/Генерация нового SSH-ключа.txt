Шаг 1: Генерация нового SSH-ключа
Откройте терминал.

Выполните команду для генерации нового SSH-ключа:

bash
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
Замените your_email@example.com на ваш реальный адрес электронной почты, связанный с вашим аккаунтом GitHub.

Вас попросят ввести имя файла для ключа. По умолчанию это ~/.ssh/id_rsa. Нажмите Enter, чтобы подтвердить путь по умолчанию.

Установите пароль для SSH-ключа (если хотите):

Вас попросят ввести пароль для ключа. Это не обязательно, но рекомендуется для повышения безопасности. Если не хотите устанавливать пароль, просто нажмите Enter.

Шаг 2: Добавление SSH-ключа в агент SSH
Убедитесь, что агент SSH запущен:

bash
eval "$(ssh-agent -s)"
Эта команда запустит агент SSH и выведет его PID (идентификатор процесса).

Добавьте ваш SSH-ключ в агент SSH:

bash
ssh-add ~/.ssh/id_rsa
Шаг 3: Копирование SSH-ключа
Скопируйте содержимое вашего публичного SSH-ключа:

bash
cat ~/.ssh/id_rsa.pub
Скопируйте весь вывод команды в буфер обмена. Это ваш публичный ключ, который необходимо добавить в ваш аккаунт GitHub.

Шаг 4: Добавление SSH-ключа в аккаунт GitHub
Перейдите на страницу настроек SSH-ключей в GitHub:

Откройте GitHub и войдите в свой аккаунт.

Перейдите в Settings -> SSH and GPG keys -> New SSH key.

Добавьте новый SSH-ключ:

В поле Title введите имя для вашего ключа (например, "My VM Key").

В поле Key вставьте содержимое вашего публичного ключа, который вы скопировали на предыдущем шаге.

Нажмите Add SSH key.

Шаг 5: Проверка подключения к GitHub
Проверьте, что ваш SSH-ключ работает:

bash
ssh -T git@github.com
Вы должны увидеть сообщение о том, что подключение успешно установлено.

Шаг 6: Клонирование репозитория
Теперь вы можете клонировать репозитории с GitHub, используя SSH:

bash
git clone git@github.com:username/repository.git
Замените username на ваше имя пользователя на GitHub, а repository на имя репозитория, который хотите клонировать.
