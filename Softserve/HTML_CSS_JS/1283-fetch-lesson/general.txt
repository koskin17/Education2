1. В проекте используется loval storage. Это небольшая база данных на стороне браузера, которую можно записывать через функции Set Item и Get item.
Увидеть её можно в браузере: Developer Tools / Application и Storage / Local Storage

Карточки товаров в online-shop должны загружаться с сервера, а не лежать в списке / массиве.
Для этого создаётся папка api (application programming interface).
API называется вообще всё внешнее, с чем мы взаимодействуем.
В backend api часто называет api end point или точка, куда надо делать запрос.
При обращении не важно знать, как называется файл. Важно знать в каком формате. При этом файл может генерировать данные на лету, а может быть так, что данные сразу записаны в файл.
С FrontEnd приходит запрос на endpoint и нам приходят данные. 
В ней создаётся файл products.json и в нём из массива карточек надо создать JSON.
Для этого можно использовать сайт jsonformater (https://jsonformatter.curiousconcept.com/)
Туда вставляет обычный js-объект или js-массив (в нашем случае массив карточек, в котором есть объекты, который был в index.product-list.js).
Полезно делать именно так потому что одна ошибка, в одном символе и JS не сможет правильно распознать JSON и не будет работать.
